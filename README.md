# 🌌 Unity DOTS 黑洞星云模拟 (v0.4.0)
-Gemini生成的

这是一个基于 Unity DOTS (ECS) 框架构建的个人练习项目。本项目主要尝试在海量实体（约 20 万粒子）的情况下，探索底层物理演算性能与前端视觉表现之间的平衡，并初步实现了一套沉浸式的视听交互体验。目前项目仍在持续学习和优化中。

## ✨ 主要功能与技术探索

### 1. 海量粒子物理演算 (DOTS + Burst)
* **引力场模拟**：利用 Job System 配合 Burst Compiler，实现了 20 万颗粒子围绕黑洞奇点的引力场与视界线物理计算。
* **零 GC 运行**：底层核心系统（如 `CalculateSingularityGravityJob`）全程采用纯值类型与非托管内存，运行期间物理演算部分保持 0 GC Allocation，初步规避了垃圾回收导致的帧数尖峰（Spike）卡顿。

### 2. 渲染表现与性能取舍
* **深度渲染优化**：针对海量粒子的 Overdraw（过度绘制）瓶颈，尝试将粒子材质从 Transparent（半透明）切换为 Opaque（不透明）并开启深度写入。利用 GPU 的 Early-Z 剔除机制，在保证画面表现力的同时大幅降低了显卡负担，并顺带解决了海量粒子深度排序计算带来的闪烁 Bug。
* **极低带宽色彩同步**：抛弃了逐粒子传递数据的做法，借助 `Shader.SetGlobalColor` 向 GPU 广播全局色彩配置。实现了通过 UI 实时改变 20 万颗粒子的极速/慢速混合颜色，且在此过程中几乎产生 0 带宽传输开销。

### 3. 沉浸式 UI 与交互细节
* **环境自适应隐藏**：引入了沉浸式状态机，当鼠标静置一段时间后，UI 面板与系统光标会自动柔和淡出，将屏幕完全交还给星云视觉；当检测到物理移动时则以倍速瞬间唤醒。
* **健壮的参数控制**：为系统参数（黑洞质量、粒子数量、引力范围等）编写了通用的 UI 同步组件。实现了 Slider 滑条与 TextMeshPro InputField 的双向数据绑定，支持精准数值输入，并内置了极值保护与错误输入拦截。
* **基础系统调度**：支持通过 UI 按钮无缝切换系统全屏，并支持检测 ESC 快捷键安全退出程序。

### 4. 底层音频流管理
* **无缝循环 BGM**：为了提供更好的氛围感，没有使用传统的 `Update` 轮询播放状态，而是尝试调用 Unity 底层 `AudioSettings.dspTime` API。在音频采样率级别精确预排期，实现了音乐“引子段落”向“循环段落”的 0 毫秒误差完美无缝拼接。
